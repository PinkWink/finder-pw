Finder — 4분할 파일 탐색기 작업 히스토리

우분투 기본 탐색기를 대체할, 화면을 최대 4분할로 나누고 각 패널에 색상·이름을 지정할 수 있는 데스크톱 탐색기 개발 기록
날짜 2026-05-07
작업 디렉토리 /home/pw/apps_with_claude/finder
스택 Tauri 2 + React 18 + TypeScript

개요

사용자는 우분투 기본 파일 매니저(Nautilus)의 화면 분할 미지원이라는 한계를 해결하기 위해, 최대 4개의 분할창을 가지고 각 분할창에 색상과 이름을 부여해 작업 효율을 높일 수 있는 탐색기 도구를 만들고자 했다. 기술 스택 비교 후 Tauri + React 기반으로 진행하기로 결정.

시간순 대화 기록

2026-05-07 12:04 KST · 사용자
초기 요구사항 제시
난 일종의 탐색기 같은걸 만들고 싶어. 그런데, 지금 기본 우분투의 탐색기는 마음에 안드는게, 화면 분할이 안되. 화면을 최대한 4개까지는 분할하고, 분할된 탐색기 각각에 색상이나 이름을 지정해서 작업의 효율을 높이고 싶어

지시 이해

  • 핵심 요구: 우분투용 데스크톱 파일 탐색기를 신규 개발
  • 차별 기능 1: 화면을 최대 4개까지 분할해 동시에 여러 디렉토리를 보기
  • 차별 기능 2: 각 분할 패널에 색상과 별칭(이름) 부여 → 시각적으로 작업 컨텍스트 분리
  • 목적: 멀티태스킹 작업 효율 향상 (단순 보기가 아닌 실용 도구)
  • 표면적 제약: 우분투 환경, 기본 탐색기보다 빠르고 가벼우면 좋음 (명시되진 않았지만 추론)

백엔드 (Rust) 핵심

  • list_directory(path) — 폴더 우선, 이름순 정렬
  • get_home_dir() — 초기 경로
  • get_parent_dir(path) — 상위 폴더 이동
  • open_file(path) — xdg-open으로 기본 앱 열기

프론트 (React) 핵심

  • 4개 패널 상태를 App.tsx에서 단일 소스로 관리
  • localStorage에 패널 구성 + 레이아웃 영속화
  • 패널 헤더에 색상 도트 + 별칭 + 경로 + ↑/⟳/⚙ 버튼
  • 설정 모달: 8색 프리셋 + 컬러 피커 + 직접 경로 입력
  • 파일 더블클릭 → 폴더면 진입, 파일이면 OS 기본 앱 실행

실행 방법

현재 시스템에는 Rust/Node 툴체인이 설치되어 있지 않습니다. 아래 순서로 진행하세요.

  1. Rust 설치: curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
  2. Node 설치 (예: nvm): curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.1/install.sh | bash 후 nvm install --lts
  3. Tauri 시스템 의존성 (우분투): sudo apt install libwebkit2gtk-4.1-dev build-essential libxdo-dev libssl-dev libayatana-appindicator3-dev librsvg2-dev
  4. 프로젝트 디렉토리에서: npm install → npm run tauri dev

첫 실행 시 Rust 컴파일에 수 분이 걸릴 수 있습니다. 이후부터는 빠릅니다.
